# 건축 현상설계 공고 큐레이션 명세

이 파일은 자동 큐레이션 작업의 단일 기준 문서. 스케줄 작업이 매 회차 이 명세를 따른다.
변경 시 이 파일만 수정하면 다음 회차부터 반영됨.

---

## 0. 사용자 도메인

- 건축가 / 현상설계·설계공모 실무자
- 관심: **이번 주에 새로 열린 공고 중 실제로 응모를 검토할 만한 것** — 마감·발주처·지역·규모(연면적/공사비/설계비)가 명확한 것
- 본인 사무소가 단독 또는 컨소시엄으로 응모 가능한 규모 신호를 중시
- 단순 뉴스·시상식 후기 ❌ / 응모 가능한 실제 공고 ✅

## 1. 절대 필터 — "응모를 검토할 수 있는 실제 공고만"

각 큐레이션 아이템은 다음 중 하나여야 함:
- 건축 설계용역 공모 / 현상설계 본 공고 (마감·발주처 있는 것)
- 건축상 / 계획·아이디어 공모전 (접수 중 또는 임박)
- 결과 발표 / 당선작 공개 (흐름 파악용 — 적은 비중)
- 접수예정·사전공고 (다음 주 대비)

자동 제외:
- 마감·발주처·규모가 불명확한 generic 항목, 포털 랜딩페이지·메뉴
- 건축과 무관한 공모 (그래픽·미술·논문·UCC·영상 등)
- 단순 시상식 뉴스·세미나·전시 안내 (응모 행위가 없는 것)
- 이미 마감이 한참 지난 종료 공고 (단, 그 주 결과 발표는 "result"로 OK)

## 2. 소스 (scripts/sources.json)

- 주택폴리오 공모전 DB (jootek.com) — 신호 가장 강함. 학교·관청·민간 폭넓게.
- 조달청 건축HUB (hub.go.kr) — 마감·발주처·위치 직접 노출.
- 위비티 건축 카테고리 (wevity.com) — 지자체 건축상·민간 공모 전국 인덱싱.
- 부산 건축설계공모 (making.busan.go.kr) — 부산권 보강.
- 한국건축가협회 게시판 (kia.or.kr) — 협회 주관 + 지자체 건축상.

소스 추가·교체는 `scripts/sources.json` 만 수정 (`kind:"html"` 기본).

## 3. 등급 (score 1~10)

요한님 실무 관점의 **응모 주목도**. 종합 판단:
- 규모·유형이 본인 사무소 응모 가능 범위에 맞음 (가산)
- 마감까지 여유가 있어 실제 준비 가능 (가산)
- 같은 유형이 그 주에 여러 건 겹쳐 한 안을 변형 응모하기 좋음 (가산)
- 설계비가 낮거나 자격 제한이 빡빡해 실익 적음 (감산)

- ★ 9~10 — 지금 당장 응모 검토할 직격 공고
- ★ 6~8 — 유형·규모 적합, 검토 가치 있음
- ★ 3~5 — 흐름상 알 가치, 직접 응모는 모호
- ★ 1~2 — 참고용 (결과 발표 등)

## 4. 출력 분량·섹션

- **headline (이번 주 핵심 공고)**: 3~5개 — 출처 무관, 그 주 응모 가치 톱. 다른 섹션과 중복 금지.
- **competition (설계공모·현상설계)**: 2~6개
- **award (건축상·아이디어공모)**: 2~6개
- **result (결과·당선작)**: 0~4개 (적은 비중)
- **notice (예정·기타)**: 0~4개
- 전체 12~22개. 해당 유형 없으면 섹션 비움 (지어내기 금지).

## 5. 언어 규칙

- 해요체/합니다체 섞어 간결하게. 과장·영업체 금지. 사실 위주.
- 건축 약어(RFP·SD·DD·연면적·용적률·건폐율 등)는 그대로 OK — 풀이 불필요.
- 발주처·지역·마감·규모는 **빠짐없이**. 수치는 원문 그대로.
- 슬랭("박다"·"꽂다") 금지 — 공개 콘텐츠.

### 항목 구성 (.ni 카드 — news·curation 과 동일 폼)

- **body** (3~4문장): 무슨 공고고 어디·무엇을 짓는지, 발주처 의도, 유사 패턴 비유 한 줄.
- **points** (불릿 2~3): 지역·마감일 / 규모(연면적·공사비·설계비) / (선택) 자격·방식.
- **시그널** (gain): 이 공고가 가리키는 흐름·패턴 — 같은 유형 몇 건인지, 지역 집중 등.
- **응모 검토** (todo): 마감·규모·자격·준비 전략. 즉시 판단에 필요한 실무 정보 (마감일·규모 수치 포함).
- **왜 지금** (why_now): 타이밍·긴급도(D-day).

### 톤 점검 셀프 질문 (작성 후)

1. 이 항목을 읽고 *건축가가 "응모할지 말지"를 1초 만에 판단*할 수 있나?
2. 마감·발주처·규모가 빠짐없이 들어갔나?
3. '시그널'이 이 공고 하나가 아니라 *그 주 흐름*을 짚고 있나?

→ 1, 2, 3 모두 ✓ 아니면 다시 쓴다.

## 6. 파일 출력

- 경로: `arch-trends-curation/YYYY-MM-DD_요일.md` (.ni 시맨틱 HTML)
- 요일: 한글 한 글자
- frontmatter: title / eyebrow / hero_title / description / summary
- 섹션 순서: 이번 주 흐름(intro) → headline → competition → award → result → notice → 맺음말(outro)
- `index.md` 는 워커가 매 회차 자동 갱신

## 7. 일정

- **현재 운영:** 매주 월요일 09:00 KST (claude -p 로컬 생성, 정액제)
- 빈도 변경 시: 이 파일과 스케줄 작업(run-daily 미러) 둘 다 수정

## 8. 중복 방지

- 직전 2회차의 최신 파일 2개를 읽고, 동일 공고 URL은 새 파일에서 제외
- 단, 같은 공고가 결과 발표 등 의미 있는 진전이 있으면 "result" 로 재등재 가능

## 9. 원문 대조 검증

- 생성 후 각 항목 URL을 실제 fetch → 마감일·연면적·공사비·설계비·발주처·지역을 원문과 대조해 정정
- 원문 못 가져온 항목은 `verified=false` (✓ 원문 대조 배지 미표시)
